Git คืออะไร? — แนวคิดหลัก

Version Control System ที่ track การเปลี่ยนแปลงโค้ด

📸 Git คือกล้องถ่ายรูปโค้ด

🗂️ 4 พื้นที่ของ Git

📁 Working Directory
ไฟล์ที่กำลังแก้ไขอยู่ — ยังไม่ได้บันทึกใน Git

ติดตั้งและตั้งค่า Git

ติดตั้ง, config identity และสร้าง SSH key

📦 ติดตั้ง Git

terminal
# Ubuntu / Debian
sudo apt update && sudo apt install git

# macOS (Homebrew)
brew install git

# Windows — ดาวน์โหลดจาก git-scm.com
winget install Git.Git

⚙️ ตั้งค่า Identity

# ─── ต้องทำทุกเครื่อง (global) ──────
git config --global user.name "ชื่อของคุณ"
git config --global user.email "email@example.com"

# ตั้ง default editor
git config --global core.editor "code --wait"

# ตั้ง default branch เป็น main
git config --global init.defaultBranch main

# ดูการตั้งค่าทั้งหมด
git config --list
git config --list --show-origin   # ดูว่ามาจากไฟล์ไหน

🔐 ตั้งค่า SSH Key สำหรับ GitHub

# สร้าง SSH key (แนะนำ ed25519)
ssh-keygen -t ed25519 -C "email@example.com"
# กด Enter ทุกขั้นตอน หรือตั้ง passphrase

# เพิ่ม key เข้า ssh-agent
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# คัดลอก public key
cat ~/.ssh/id_ed25519.pub
# นำ key ไปเพิ่มใน GitHub → Settings → SSH keys

# ทดสอบการเชื่อมต่อ
ssh -T git@github.com
Hi username! You've successfully authenticated. 🎉

คำสั่งพื้นฐาน

init, add, commit, status, log — วงจรชีวิตของ commit

🔄 วงจรการทำงานพื้นฐาน (Daily Workflow)

# 1. สร้าง repository ใหม่
mkdir my-project && cd my-project
git init
Initialized empty Git repository in .git/

# 2. ดูสถานะ
git status
On branch main
nothing to commit, working tree clean

# 3. เพิ่มไฟล์เข้า staging
git add index.html          # เฉพาะไฟล์
git add src/                # ทั้ง directory
git add *.js                # glob pattern
git add .                   # ทุกไฟล์ใน dir นี้
git add -A                  # ทุกไฟล์ทั้งโปรเจกต์
git add -p                  # เลือก staging ทีละส่วน

# 4. Commit
git commit -m "feat: add login page"
git commit -am "fix: typo in README"  # add + commit (tracked files)

# 5. ดูประวัติ
git log                     # ดูทั้งหมด
git log --oneline           # แบบย่อ
git log --oneline --graph   # แสดง branch graph

Branch — สาขาการทำงาน

สร้าง, เปลี่ยน, และจัดการ branches

🌿 Branch คืออะไร

Branch คือ pointer ที่ชี้ไปยัง commit หนึ่ง — การสร้าง branch ใหม่ไม่ copy โค้ด แค่สร้าง pointer ใหม่ ทำให้เร็วมาก

main
C1
C2
C3
C6
C7
← HEAD
feature
C3
C4
C5
← merged to main

🌿 จัดการ Branch

# ดู branches ทั้งหมด
git branch              # local
git branch -r           # remote
git branch -a           # ทั้งหมด

# สร้าง branch
git branch feature/login

# สร้างและเปลี่ยนไปเลย (แนะนำ)
git checkout -b feature/login
git switch -c feature/login    # ใหม่กว่า

# เปลี่ยน branch
git checkout main
git switch main              # ใหม่กว่า

# เปลี่ยนชื่อ branch
git branch -m old-name new-name

# ลบ branch
git branch -d feature/login   # merged แล้ว
git branch -D feature/login   # บังคับลบ

🔍 ดูความแตกต่าง (diff)

# working dir vs staging
git diff

# staging vs last commit
git diff --staged
git diff --cached

# ระหว่าง branches
git diff main..feature/login
git diff main feature/login -- src/

# ระหว่าง commits
git diff abc123..def456

# ดูไฟล์ที่เปลี่ยนไป
git diff --name-only main

# ดูไฟล์ใน commit
git show abc123
git show abc123:src/app.js  # ดูไฟล์ใน commit
05

Merge & Rebase

รวมโค้ดจาก branches ต่างๆ และจัดการ conflicts

🔀 Merge (เก็บ history)
# อยู่บน main
git switch main
git merge feature/login

# สร้าง merge commit
Merge branch 'feature/login'

─ ผลลัพธ์ ─
main:  C1─C2─C3─C5(merge)
                ↗
feat:  C1─C2─C4
⛓️ Rebase (history สะอาด)
# อยู่บน feature
git switch feature/login
git rebase main

# ย้าย commits ต่อจาก main
Successfully rebased and updated refs/heads/feature

─ ผลลัพธ์ ─
main:  C1─C2─C3
feat:  C1─C2─C3─C4' (rebased)
       ← เส้นตรง ไม่มี merge commit

⚠️ Merge Conflict — วิธีแก้

# เกิด conflict ตอน merge
git merge feature/login
CONFLICT (content): Merge conflict in src/app.js
Automatic merge failed; fix conflicts and then commit.

# ดูไฟล์ที่ conflict
git status
both modified: src/app.js

# ใน src/app.js จะมี conflict markers:
<<<<<<< HEAD (main branch)
const PORT = 3000
=======
const PORT = 8080
>>>>>>> feature/login

# แก้ไขไฟล์ → ลบ markers เลือกโค้ดที่ถูก:
const PORT = 8080

# หลังแก้ conflict ทุกไฟล์
git add src/app.js
git commit -m "merge: resolve conflict in app.js"

# หรือยกเลิก merge
git merge --abort
💡
ใช้ VS Code, IntelliJ หรือ git mergetool เปิด visual conflict editor ได้สะดวกกว่าแก้ manual

⛓️ Rebase Interactive — จัดการ commits

# rebase 3 commits ล่าสุด
git rebase -i HEAD~3

# เปิด editor แสดง:
pick a1b2c3 feat: add login
pick d4e5f6 fix: typo
pick g7h8i9 fix: another typo

# เปลี่ยน pick เป็น command ต่างๆ:
# pick   = ใช้ commit นี้
# reword = แก้ commit message
# squash = รวมกับ commit ก่อนหน้า (เก็บ msg)
# fixup  = รวมกับ commit ก่อนหน้า (ทิ้ง msg)
# drop   = ลบ commit นี้ทิ้ง
# edit   = หยุดเพื่อแก้ไข

# รวม 2 fix เข้ากับ feat:
pick a1b2c3 feat: add login
fixup d4e5f6 fix: typo
fixup g7h8i9 fix: another typo
⚠️
Golden Rule: อย่า rebase commits ที่ push ไป remote แล้ว — จะทำให้ history ของคนอื่นเสีย

Remote — ทำงานกับ GitHub

clone, push, pull, fetch — เชื่อมต่อ remote repository

☁️ Remote Commands

# ─── Clone ───────────────────────────────────────────
git clone https://github.com/user/repo.git
git clone git@github.com:user/repo.git       # SSH
git clone https://... myapp                  # ตั้งชื่อ dir
git clone --depth 1 https://...              # shallow clone

# ─── Remote ──────────────────────────────────────────
git remote -v                                # ดู remotes
origin  git@github.com:user/repo.git (fetch)
origin  git@github.com:user/repo.git (push)

git remote add origin git@github.com:u/r.git # เพิ่ม remote
git remote rename origin upstream             # เปลี่ยนชื่อ
git remote remove upstream                    # ลบ remote
git remote set-url origin git@github.com:...  # เปลี่ยน URL

# ─── Push ────────────────────────────────────────────
git push origin main               # push branch นี้
git push -u origin main            # -u = set upstream (ครั้งแรก)
git push                             # หลังจาก -u ก็ใช้ git push ได้เลย
git push origin feature/login      # push feature branch
git push --tags                      # push tags
git push origin --delete old-branch # ลบ remote branch

⬇️ fetch vs pull

# fetch — ดึงมาแต่ไม่ merge
git fetch origin          # ดึงทุก branch
git fetch origin main    # เฉพาะ main
git fetch --all           # ทุก remote

# หลัง fetch ดูความต่าง
git diff main origin/main
git log main..origin/main

# pull = fetch + merge
git pull                  # merge (default)
git pull --rebase         # rebase แทน merge (สะอาดกว่า)
git pull origin main

# ตั้งค่า pull เป็น rebase เสมอ
git config --global pull.rebase true

📋 Tracking Branches

# ดู tracking relationship
git branch -vv
* main       a3f9c12 [origin/main] fix: bug
  feature    b1e4d89 [origin/feature: ahead 2]

# ตั้ง upstream ให้ branch
git branch --set-upstream-to=origin/main main

# pull ด้วย strategy
git pull --ff-only   # fast-forward เท่านั้น

# ดู remote branches
git branch -r
origin/main
origin/develop
origin/feature/auth

Undo & Reset — ย้อนกลับ

restore, reset, revert — เลือกให้ถูกสถานการณ์

🚨
ระวัง! — git reset --hard และ git clean -fd ทำลายข้อมูลโดยไม่มี undo ตรวจสอบก่อนรันเสมอ

🔄 git restore — แก้ Working Directory & Staging

# ─── ทิ้งการแก้ไขใน Working Directory ─────────────────
git restore index.html        # คืนค่าจาก last commit
git restore .                  # คืนทั้งหมด
git restore *.js               # pattern

# ─── เอาออกจาก Staging Area ────────────────────────────
git restore --staged index.html
git restore --staged .

# ─── คืนค่าจาก commit อื่น ──────────────────────────────
git restore --source HEAD~2 index.html  # จาก 2 commits ก่อน
git restore --source abc123 src/       # จาก commit hash

⏪ git reset — ย้าย HEAD

# git reset [mode] [commit]

──── --soft ────────────────────────
ย้าย HEAD เท่านั้น
Working Dir: ไม่เปลี่ยน
Staging:     ไม่เปลี่ยน
git reset --soft HEAD~1
# commit หายแต่ไฟล์ยังอยู่ใน staging
# ✅ ใช้เมื่อ: แก้ commit message หรือรวม commits

──── --mixed (default) ─────────────
ย้าย HEAD + unstage
Working Dir: ไม่เปลี่ยน
Staging:     reset
git reset HEAD~1
git reset --mixed HEAD~1
# ✅ ใช้เมื่อ: staged ผิดไฟล์
──── --hard ────────────────────────
ย้าย HEAD + reset ทุกอย่าง
Working Dir: ลบทิ้ง!
Staging:     ลบทิ้ง!
git reset --hard HEAD~1
git reset --hard abc123
# ⚠️ ข้อมูลหายถาวร (ยกเว้น reflog)
# ✅ ใช้เมื่อ: ล้างทุกอย่างแน่ๆ

──── กู้คืนด้วย reflog ─────────────
git reflog
a3f9c12 HEAD@{0}: reset: moving to HEAD~1
b1e4d89 HEAD@{1}: commit: feat: login

# กู้คืน
git reset --hard b1e4d89
# ✅ ย้อนกลับได้ภายใน 90 วัน

↩️ git revert — ปลอดภัยกว่า reset

# revert สร้าง commit ใหม่ที่ "undo" commit เก่า
# ✅ ปลอดภัย — ไม่ลบ history ใช้ได้กับ shared branches

git revert abc123           # revert commit นั้น
git revert HEAD              # revert commit ล่าสุด
git revert HEAD~3..HEAD      # revert 3 commits
git revert --no-commit abc123  # revert ไม่ต้อง commit ทันที

# ต่างจาก reset อย่างไร:
reset:  A─B─C  →  A─B       (C หาย)
revert: A─B─C  →  A─B─C─C'  (C' undo C)
Commandเปลี่ยน Historyปลอดภัยบน Shared Branchใช้เมื่อ
git restore❌✅ทิ้งการแก้ไขที่ยังไม่ commit
git reset --soft✅⚠️ local onlyแก้ commit ล่าสุด
git reset --hard✅❌ทิ้งทุกอย่างแน่ๆ
git revert❌ (เพิ่ม commit)✅undo บน branch ที่ share แล้ว

Stash — เก็บงานชั่วคราว

เมื่อต้องเปลี่ยน branch แต่งานยังไม่เสร็จ

📦 git stash

# บันทึกงานปัจจุบันลง stash
git stash                          # stash tracked files
git stash push -m "WIP: login form"  # ตั้งชื่อ stash
git stash -u                       # รวม untracked files
git stash --all                    # รวม ignored files ด้วย

# ─── ดู stash list ───────────────────────────────────
git stash list
stash@{0}: On main: WIP: login form
stash@{1}: On feature: WIP: cart

# ─── นำ stash กลับมา ────────────────────────────────
git stash pop               # apply stash ล่าสุด + ลบออก
git stash apply             # apply ล่าสุด แต่ไม่ลบ
git stash apply stash@{1}   # apply stash ที่ระบุ
git stash pop stash@{1}     # pop stash ที่ระบุ

# ─── จัดการ stash ───────────────────────────────────
git stash drop stash@{0}    # ลบ stash
git stash clear             # ลบทั้งหมด
git stash show -p           # ดู diff ของ stash
git stash branch feature/new # สร้าง branch จาก stash

Advanced Commands

cherry-pick, tag, bisect, worktree — คำสั่งขั้นสูง

🍒 Cherry-pick — เลือก commit

# นำ commit จาก branch อื่นมาใช้
git cherry-pick abc123
git cherry-pick abc123 def456   # หลาย commits
git cherry-pick abc123..def456 # range
git cherry-pick --no-commit abc123  # ไม่ commit

# เมื่อ conflict:
# 1. แก้ conflict
# 2. git add .
# 3. git cherry-pick --continue
# หรือ git cherry-pick --abort

🏷️ Tag — เวอร์ชัน

# สร้าง tag
git tag v1.0.0                    # lightweight
git tag -a v1.0.0 -m "Release 1.0"  # annotated
git tag -a v1.0.0 abc123         # tag เก่า

# ดู tags
git tag
git tag -l "v1.*"
git show v1.0.0

# push tags
git push origin v1.0.0
git push origin --tags

# ลบ tag
git tag -d v1.0.0
git push origin --delete v1.0.0

🔍 git bisect — หา commit ที่ทำให้ bug เกิด

# Binary search หา commit ที่ทำให้เกิด bug
git bisect start
git bisect bad                 # HEAD ปัจจุบันมี bug
git bisect good v1.0.0         # เวอร์ชันนี้ยังดีอยู่

# Git จะ checkout commit กลาง → ทดสอบ → บอกว่าดีหรือไม่
git bisect good                # commit นี้ยังดี
git bisect bad                 # commit นี้มี bug

# ทำซ้ำจนกว่า Git จะหาเจอ
abc1234 is the first bad commit
Author: สมชาย  Date: Mon Jan 1

git bisect reset               # กลับมาที่ HEAD

🎨 git aliases — คำสั่งลัด

# ตั้งค่า aliases ใน ~/.gitconfig
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit

# Aliases ที่มีประโยชน์
git config --global alias.lg "log --oneline --graph --all --decorate"
git config --global alias.last "log -1 HEAD --stat"
git config --global alias.undo "reset --soft HEAD~1"
git config --global alias.unstage "restore --staged"
git config --global alias.amend "commit --amend --no-edit"

# ใช้งาน
git lg             # แสดง graph สวย
git undo           # ย้อน commit ล่าสุด
git unstage .      # unstage ทั้งหมด

develop ถาวร— รวม features ที่เสร็จแล้ว รอ release
feature/xxx ชั่วคราวdevelop พัฒนา feature ใหม่
release/x.x ชั่วคราวmain + develop เตรียม release — bug fix เท่านั้น
hotfix/xxx ชั่วคราวmain + develop แก้ critical bug บน production ด่วน

⚙️ Git Flow Commands

# ─── Feature ──────────────────────────────────────────
git switch develop
git switch -c feature/user-auth

# ... พัฒนา feature ...

git switch develop
git merge --no-ff feature/user-auth  # --no-ff = เก็บ merge commit
git branch -d feature/user-auth

# ─── Release ──────────────────────────────────────────
git switch develop
git switch -c release/1.2.0

# ... แก้ bug เล็กน้อย / อัปเดต version ...

git switch main
git merge --no-ff release/1.2.0
git tag -a v1.2.0 -m "Version 1.2.0"
git switch develop
git merge --no-ff release/1.2.0
git branch -d release/1.2.0

# ─── Hotfix ───────────────────────────────────────────
git switch main
git switch -c hotfix/critical-bug

# ... แก้ bug ...

git switch main
git merge --no-ff hotfix/critical-bug
git tag -a v1.2.1 -m "Hotfix 1.2.1"
git switch develop
git merge --no-ff hotfix/critical-bug
git branch -d hotfix/critical-bug

GitHub Workflow

Fork, Pull Request, Code Review และ GitHub Actions

🍴 Fork & Pull Request Workflow

1
Fork repo
→ สร้าง copy ของ repo ใน account ตัวเอง
2
git clone fork
→ clone fork มายังเครื่องตัวเอง
3
branch + commit
→ สร้าง feature branch ทำงาน + commit
4
git push
→ push branch ขึ้น fork
5
Pull Request
→ เปิด PR บน GitHub ขอ merge เข้า upstream
6
Code Review
→ ทีม review → approve / request changes
7
Merge PR
→ merge เข้า main/develop ✅

🔄 Sync Fork กับ Upstream

# เพิ่ม upstream remote
git remote add upstream https://github.com/original/repo.git

# ดู remotes
git remote -v
origin    git@github.com:you/repo.git
upstream  git@github.com:original/repo.git

# sync กับ upstream
git fetch upstream
git switch main
git merge upstream/main

# หรือ rebase
git rebase upstream/main

# push sync ขึ้น fork
git push origin main

🔁 GitHub Actions (CI/CD)

# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm test
      - run: npm run build

Cheatsheet — คำสั่งทั้งหมด

สรุปคำสั่ง Git ที่ใช้บ่อยในที่เดียว

📁 Setup & Init
# ตั้งค่า
git config --global user.name "name"
git config --global user.email "email"

# สร้าง / clone
git init
git clone <url>
📸 Stage & Commit
git status             # ดูสถานะ
git add .              # stage ทั้งหมด
git add -p             # stage เลือกส่วน
git commit -m "msg"
git commit --amend     # แก้ commit ล่าสุด
git diff --staged
🌿 Branch
git branch             # ดู branches
git switch -c <name>   # สร้าง+เปลี่ยน
git switch <name>      # เปลี่ยน
git branch -d <name>   # ลบ (merged)
git branch -D <name>   # บังคับลบ
🔀 Merge & Rebase
git merge <branch>
git merge --no-ff <branch>
git rebase <branch>
git rebase -i HEAD~3
git merge --abort
git rebase --abort
☁️ Remote
git remote -v
git remote add origin <url>
git push -u origin main
git push
git fetch
git pull --rebase
⏪ Undo
git restore <file>       # ทิ้ง changes
git restore --staged <f> # unstage
git reset --soft HEAD~1  # undo commit
git reset --hard HEAD~1  # ⚠️ ลบทิ้ง
git revert <hash>        # ปลอดภัย
git reflog               # กู้คืน
📦 Stash
git stash
git stash push -m "WIP"
git stash list
git stash pop
git stash apply stash@{1}
git stash drop stash@{0}
🔍 Log & Search
git log --oneline --graph
git log --author="name"
git log -p <file>
git blame <file>
git grep "text"
git log -S "text"       # pickaxe
🍒 Advanced
git cherry-pick <hash>
git tag -a v1.0 -m "msg"
git bisect start
git bisect good/bad
git worktree add <path>
git submodule add <url>
💡
Workflow แนะนำทุกวัน:
git pull --rebase → git switch -c feature/xxx → ทำงาน → git add -p → git commit -m "feat: ..." → git push -u origin feature/xxx → เปิด Pull Request
