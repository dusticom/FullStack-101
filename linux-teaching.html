File System — สร้าง คัดลอก ย้าย ลบ

จัดการไฟล์และ directory ทุกรูปแบบ

🗂️ โครงสร้าง Linux File System

Directoryใช้เก็บอะไร
/Root — ราก ของทุกอย่าง
/homehome ของแต่ละ user เช่น /home/user
/roothome ของ root
/etcconfig files ของระบบ
/varlogs, databases, spool files
/tmpไฟล์ชั่วคราว — ถูกลบเมื่อ reboot
/usrโปรแกรมและ libraries สำหรับ user
/bin, /sbinคำสั่งพื้นฐาน (ls, cp, bash…)
/devdevice files (disk, terminal)
/proc, /sysvirtual filesystem — ข้อมูล kernel/process
/optโปรแกรม third-party ที่ติดตั้งเพิ่ม

📁 สร้าง / ลบ Directory & File

# ─── Directory ───────────────────────
$ mkdir photos          # สร้าง dir
$ mkdir -p a/b/c        # สร้างซ้อนกัน
$ rmdir photos          # ลบ dir ว่าง
$ rm -r mydir           # ลบ dir + เนื้อหา
$ rm -rf mydir          # ไม่ถาม (ระวัง!)

# ─── File ─────────────────────────────
$ touch file.txt        # สร้างไฟล์ว่าง
$ touch {a,b,c}.txt     # สร้างหลายไฟล์
$ rm file.txt           # ลบไฟล์
$ rm -i file.txt        # ถามก่อนลบ
$ rm *.log              # ลบทุก .log

📋 คัดลอก & ย้ายไฟล์

# ─── Copy ─────────────────────────────
$ cp file.txt backup.txt  # copy file
$ cp -r dir1 dir2         # copy directory
$ cp -p file.txt dest/    # รักษา permission
$ cp *.txt ~/backup/      # copy หลายไฟล์

# ─── Move / Rename ────────────────────
$ mv file.txt newname.txt # เปลี่ยนชื่อ
$ mv file.txt ~/Documents/# ย้าย
$ mv dir1 /opt/           # ย้าย dir

# ─── Link ─────────────────────────────
$ ln -s /var/log logs     # symlink
$ ln file.txt hard.txt    # hard link

🔍 find — ค้นหาไฟล์

# find [path] [options] [expression]
$ find . -name "*.txt"              # ค้นหาจากชื่อ
$ find . -iname "*.TXT"             # case insensitive
$ find /var/log -name "*.log" -mtime -7  # แก้ไขใน 7 วันที่ผ่านมา
$ find . -type f -size +10M         # ไฟล์ขนาดมากกว่า 10MB
$ find . -type d -name "node_modules"# หา directory ชื่อ node_modules
$ find . -empty                     # ไฟล์/dir ว่าง
$ find . -perm 644                  # ไฟล์ที่มี permission 644
$ find . -user john                 # ไฟล์ของ user john

# รัน command กับผลลัพธ์
$ find . -name "*.log" -delete      # ลบไฟล์ที่เจอ
$ find . -name "*.sh" -exec chmod +x {} \;  # exec command

File Permissions

rwx, chmod, chown — ควบคุมสิทธิ์การเข้าถึงไฟล์

🔑 อ่าน Permission

- rwx owner r-x group r-- others
ตำแหน่งความหมาย
- (ตัวแรก)- = file, d = directory, l = symlink
r (read)อ่านได้ — ค่า 4
w (write)เขียน/แก้ไขได้ — ค่า 2
x (execute)รันได้ (file) / เข้า dir ได้ — ค่า 1
-ไม่มีสิทธิ์นั้น

🔢 chmod — Numeric Mode

# r=4, w=2, x=1 บวกรวมกัน
# 7 = rwx  6 = rw-  5 = r-x
# 4 = r--  3 = -wx  0 = ---

$ chmod 755 script.sh   # rwxr-xr-x
$ chmod 644 file.txt    # rw-r--r--
$ chmod 600 private.key # rw-------
$ chmod 777 public/     # rwxrwxrwx (ระวัง!)
$ chmod -R 755 mydir/   # recursive

🔤 chmod — Symbolic Mode

# u=user g=group o=others a=all
# + เพิ่ม  - ลบ  = กำหนด

$ chmod +x script.sh    # ทุกคน execute
$ chmod u+x script.sh   # เฉพาะ owner
$ chmod g-w file.txt    # ลบ write จาก group
$ chmod o=r file.txt    # others อ่านได้อย่างเดียว
$ chmod a-x file.txt    # ลบ execute ทุกคน
$ chmod u+rw,g+r,o= f  # หลาย rule พร้อมกัน

👤 chown / chgrp — เปลี่ยนเจ้าของ

$ sudo chown john file.txt        # เปลี่ยน owner
$ sudo chown john:devs file.txt   # เปลี่ยน owner:group
$ sudo chown -R www-data /var/www # recursive
$ sudo chgrp docker /usr/bin/docker# เปลี่ยนแค่ group

# sudo — ทำในนาม root
$ sudo apt update    # รันเป็น root
$ sudo -i            # เปิด root shell
$ sudo -u john cmd   # รันในนาม user อื่น
PermissionNumericใช้กับ
rwxr-xr-x755executables, directories ทั่วไป
rw-r--r--644text files, config files
rw-------600private keys, sensitive files
rwx------700private scripts/directories
rwxrwxr-x775shared project directory

Text Processing

cat, less, grep, sed, awk, cut, sort — เครื่องมือจัดการข้อความ

📖 อ่านไฟล์

$ cat file.txt       # แสดงทั้งหมด
$ cat -n file.txt    # แสดงพร้อมเลขบรรทัด
$ cat a.txt b.txt    # แสดงต่อกัน

$ head -20 file.txt  # 20 บรรทัดแรก
$ tail -20 file.txt  # 20 บรรทัดสุดท้าย
$ tail -f app.log    # ติดตาม log แบบ real-time

$ less file.txt      # อ่านทีละหน้า
# ใน less: Space=ลงหน้า b=ขึ้นหน้า
#          /word=ค้นหา n=ถัดไป q=ออก

$ wc file.txt        # นับ lines words bytes
$ wc -l file.txt     # นับแค่บรรทัด

✏️ เขียนลงไฟล์

# > เขียนทับ (overwrite)
$ echo "hello" > file.txt
$ echo "world" > file.txt  # ทับของเดิม!

# >> ต่อท้าย (append)
$ echo "line 1" > file.txt
$ echo "line 2" >> file.txt

# heredoc — เขียนหลายบรรทัด
$ cat > config.txt << 'EOF'
HOST=localhost
PORT=3000
DEBUG=true
EOF

# tee — แสดงบนหน้าจอ + บันทึกไฟล์
$ ls -la | tee output.txt

🔍 grep — ค้นหาข้อความ

# grep [options] pattern [file]
$ grep "error" app.log           # ค้นหา "error"
$ grep -i "ERROR" app.log         # case insensitive
$ grep -n "error" app.log         # แสดงเลขบรรทัด
$ grep -v "DEBUG" app.log         # บรรทัดที่ไม่มี DEBUG
$ grep -r "TODO" ./src            # ค้นใน dir (recursive)
$ grep -l "error" *.log           # แค่ชื่อไฟล์
$ grep -c "error" app.log         # นับจำนวนบรรทัด
$ grep -A3 "FATAL" app.log        # 3 บรรทัดหลัง match
$ grep -B2 "FATAL" app.log        # 2 บรรทัดก่อน match
$ grep -E "error|warning" app.log # regex extended
$ grep "^ERROR" app.log           # ขึ้นต้นด้วย ERROR
$ grep "\.js$" file.txt           # ลงท้ายด้วย .js

⚙️ sed — แทนที่ข้อความ

# sed 's/old/new/' — substitute
$ sed 's/foo/bar/' file.txt       # แทนที่แค่อันแรก
$ sed 's/foo/bar/g' file.txt      # แทนที่ทุกอัน (g=global)
$ sed 's/foo/bar/gi' file.txt     # case insensitive
$ sed -i 's/foo/bar/g' file.txt   # แก้ไขไฟล์ตรง (in-place)
$ sed -i.bak 's/foo/bar/g' f.txt  # backup เป็น f.txt.bak

# ลบบรรทัด
$ sed '/pattern/d' file.txt       # ลบบรรทัดที่มี pattern
$ sed '5d' file.txt               # ลบบรรทัดที่ 5
$ sed '1,3d' file.txt             # ลบบรรทัด 1-3

# แสดงเฉพาะบางบรรทัด
$ sed -n '5,10p' file.txt         # แสดงบรรทัด 5-10
$ sed -n '/start/,/end/p' file.txt# ระหว่าง pattern

📊 awk — ประมวลผลข้อมูล Column

# $1 $2 $3 = column 1 2 3, $NF = column สุดท้าย
$ awk '{print $1}' file.txt          # column แรก
$ awk '{print $1, $3}' file.txt       # column 1 และ 3
$ awk -F: '{print $1}' /etc/passwd    # -F = delimiter (:)
$ awk '{sum+=$1} END{print sum}' f    # รวมค่า column 1
$ awk 'NR==5' file.txt               # บรรทัดที่ 5
$ awk 'NR>=5 && NR<=10' file.txt     # บรรทัด 5-10
$ awk '/error/{print NR, $0}' app.log # บรรทัดที่มี error
$ ls -l | awk '{print $5, $9}'        # ขนาด + ชื่อไฟล์

# sort และ uniq
$ sort file.txt                       # เรียง a-z
$ sort -r file.txt                    # เรียง z-a
$ sort -n numbers.txt                 # เรียงตัวเลข
$ sort -u file.txt                    # ไม่ซ้ำ
$ sort file.txt | uniq -c | sort -rn  # นับความถี่
$ cut -d: -f1 /etc/passwd             # ตัด column (: delimiter)

Process Management

ps, top, kill, jobs, fg/bg — ดูและจัดการ process

📊 ดู Process

$ ps              # process ของ shell นี้
$ ps aux          # process ทั้งหมดในระบบ
$ ps aux | grep nginx  # หา nginx
$ pgrep nginx     # PID ของ nginx
$ pstree          # แสดงเป็น tree

# top / htop — real-time
$ top             # monitor แบบ real-time
$ htop            # top ที่สวยกว่า (ต้องติดตั้ง)

# ใน top: q=ออก k=kill M=เรียงRAM
#         P=เรียง CPU u=กรอง user

🔫 Kill Process

# kill signal
$ kill 1234       # SIGTERM (หยุดสุภาพ)
$ kill -9 1234    # SIGKILL (บังคับหยุด)
$ kill -l         # list signals ทั้งหมด
$ pkill nginx     # kill ตามชื่อ
$ killall node    # kill ทุก process ชื่อ node

# Signals สำคัญ
# SIGTERM (15) — ขอให้หยุด gracefully
# SIGKILL (9)  — บังคับหยุดทันที
# SIGHUP  (1)  — reload config
# SIGINT  (2)  — Ctrl+C

⚙️ Background / Foreground Jobs

# รันแบบ background
$ sleep 100 &          # & = รันใน background
[1] 12345             # [job_num] PID

$ jobs                 # ดู background jobs
[1]+  Running    sleep 100 &

$ fg %1               # นำ job 1 กลับ foreground
$ bg %1               # ส่ง job 1 ไป background

# Ctrl+Z = suspend → แล้วค่อย bg
$ vim file.txt
# กด Ctrl+Z
[1]+  Stopped    vim file.txt
$ bg                   # ส่งไป background

# nohup — ทำงานต่อแม้ปิด terminal
$ nohup node server.js &
$ nohup ./script.sh > output.log 2>&1 &

Pipe & Redirection

|, >, <, 2>, &&, || — เชื่อมคำสั่งและจัดการ I/O

🔄 Standard Streams

Streamหมายเลขคือ
stdin0ข้อมูลขาเข้า (keyboard)
stdout1ข้อมูลขาออกปกติ
stderr2ข้อความ error
stdin (0)
→
COMMAND
→
stdout (1)
↘
stderr (2)

📤 Redirection

# stdout redirect
$ ls > files.txt           # เขียนทับ
$ ls >> files.txt          # ต่อท้าย

# stderr redirect
$ ls /x 2> error.txt      # error ไปไฟล์
$ ls /x 2>> error.txt     # error ต่อท้าย
$ ls /x 2> /dev/null      # ทิ้ง error

# stdout + stderr รวมกัน
$ cmd > out.txt 2>&1      # ทั้งคู่ไปไฟล์
$ cmd &> out.txt           # แบบย่อ
$ cmd > /dev/null 2>&1   # ทิ้งทั้งหมด

# stdin redirect
$ sort < names.txt         # อ่านจากไฟล์
$ mysql db < dump.sql      # import SQL

🔗 Pipe และการเชื่อมคำสั่ง

# | pipe — ส่ง stdout ของซ้ายเป็น stdin ของขวา
$ ls -la | grep ".txt"              # filter
$ cat access.log | grep "404" | wc -l# นับ 404
$ ps aux | sort -k3 -rn | head -10  # top 10 CPU
$ cat /etc/passwd | awk -F: '{print $1}' | sort

# && — รันถัดไปถ้าสำเร็จ (exit 0)
$ mkdir project && cd project && git init
$ npm install && npm start

# || — รันถัดไปถ้าล้มเหลว
$ ping server || echo "server down"

# ; — รันทุกอันไม่สนผลลัพธ์
$ echo "start"; ./script.sh; echo "done"

# xargs — แปลง stdout เป็น argument
$ find . -name "*.log" | xargs rm
$ cat urls.txt | xargs wget
$ echo "a b c" | xargs -n1 echo     # ทีละอัน

Networking Commands

ping, curl, wget, netstat, ss — ตรวจสอบและจัดการ network

🌐 ตรวจสอบ Network

$ ping google.com       # ทดสอบ connectivity
$ ping -c4 8.8.8.8      # 4 ครั้งแล้วหยุด
$ traceroute google.com # ดู route
$ nslookup google.com   # DNS lookup
$ dig google.com        # DNS detail
$ host google.com       # DNS ย่อ

# network interfaces
$ ip addr               # IP addresses (ip a)
$ ip route              # routing table
$ ifconfig              # (เก่า)

🔌 Ports และ Connections

# ss (socket statistics) — แทน netstat
$ ss -tulnp       # TCP/UDP listening ports
$ ss -an          # connections ทั้งหมด
$ ss -tnp         # TCP + process name

# netstat (เก่า แต่ยังใช้ได้)
$ netstat -tulnp  # listening ports
$ netstat -an     # all connections

# ตรวจ port เฉพาะ
$ ss -lnp | grep :3000
$ lsof -i :3000   # ใครใช้ port 3000

🌍 curl — HTTP Client

# GET
$ curl https://api.example.com/users
$ curl -s https://api.example.com       # -s = silent
$ curl -o file.html https://example.com # บันทึกลงไฟล์
$ curl -I https://example.com           # headers เท่านั้น
$ curl -L https://bit.ly/abc            # follow redirect

# POST JSON
$ curl -X POST https://api.example.com/users \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer TOKEN" \
  -d '{"name":"John","email":"john@example.com"}'

# PUT / DELETE
$ curl -X PUT  https://api.example.com/users/1 -d '{"name":"Jane"}'
$ curl -X DELETE https://api.example.com/users/1

# Download ไฟล์
$ wget https://example.com/file.zip
$ wget -O myfile.zip https://example.com/file.zip
$ wget -c https://example.com/big.iso  # resume download

Archive & Compression

tar, gzip, zip — บีบอัดและแตกไฟล์

📦 tar — Archive ยอดนิยม

# c=create x=extract f=file
# z=gzip    j=bzip2  v=verbose

# ─── สร้าง archive ──────────────────
$ tar -czf backup.tar.gz mydir/
$ tar -cjf backup.tar.bz2 mydir/   # bzip2
$ tar -cf  backup.tar mydir/        # ไม่บีบ

# ─── แตกไฟล์ ────────────────────────
$ tar -xzf backup.tar.gz            # แตกใน dir นี้
$ tar -xzf backup.tar.gz -C /tmp/  # แตกใน /tmp

# ─── ดูเนื้อหา ────────────────────
$ tar -tzf backup.tar.gz            # list ไฟล์

🗜️ gzip / zip

# gzip — บีบเดียว
$ gzip file.txt         # → file.txt.gz
$ gzip -k file.txt      # -k เก็บต้นฉบับ
$ gunzip file.txt.gz    # แตก gzip
$ zcat file.txt.gz      # อ่านโดยไม่แตก

# zip — รองรับ Windows
$ zip archive.zip f1 f2 f3
$ zip -r archive.zip mydir/  # recursive
$ unzip archive.zip
$ unzip archive.zip -d /tmp/ # แตกไปที่
$ unzip -l archive.zip       # ดูเนื้อหา

User & Group Management

useradd, passwd, groups — จัดการผู้ใช้ในระบบ

👤 จัดการ User

# สร้าง user
$ sudo useradd -m john         # -m = สร้าง home dir
$ sudo useradd -m -s /bin/bash -G sudo john

# ตั้ง / เปลี่ยน password
$ sudo passwd john
$ passwd                        # เปลี่ยนของตัวเอง

# แก้ไข user
$ sudo usermod -aG docker john  # เพิ่ม group
$ sudo usermod -s /bin/zsh john # เปลี่ยน shell
$ sudo usermod -l newname john  # เปลี่ยนชื่อ

# ลบ user
$ sudo userdel john             # ลบ user
$ sudo userdel -r john          # ลบทั้ง home dir

# ดู user info
$ id john                       # UID GID groups
$ groups john                   # groups ที่อยู่
$ who                           # ใครล็อกอินอยู่
$ last                          # ประวัติการล็อกอิน

👥 จัดการ Group

$ sudo groupadd devs       # สร้าง group
$ sudo groupdel devs       # ลบ group
$ cat /etc/group           # ดู groups ทั้งหมด
$ cat /etc/passwd          # ดู users ทั้งหมด

# sudo privileges
$ sudo usermod -aG sudo john  # Ubuntu
$ sudo usermod -aG wheel john # CentOS/RHEL
$ sudo visudo                 # แก้ sudoers

# su — switch user
$ su john       # เปลี่ยนเป็น john
$ su - john     # เปลี่ยนพร้อม environment
$ su -          # เปลี่ยนเป็น root

Shell Scripting

Variables, Conditions, Loops, Functions — เขียน script อัตโนมัติ

📝 โครงสร้าง Script พื้นฐาน

#!/bin/bash
# ↑ shebang — บอกว่าใช้ interpreter อะไร

📦 Variables

#!/usr/bin/env bash

# กำหนด — ห้ามมี space รอบ =
name="สมชาย"
age=25
greeting="สวัสดี $name"

# อ่านค่า
echo "$name"       # สมชาย
echo "${name}123" # กำกัด variable

# Readonly
readonly PI=3.14159

# Command substitution
today=$(date +%Y-%m-%d)
files=$(ls | wc -l)

# Special variables
# $0  = ชื่อ script
# $1..$9 = arguments
# $@ = arguments ทั้งหมด (array)
# $# = จำนวน arguments
# $? = exit code ของ command ล่าสุด
# $$ = PID ของ script นี้
echo "Script: $0, Args: $#"

🔀 Conditions

#!/usr/bin/env bash

score=85

# if / elif / else
if [[ $score -ge 90 ]]; then
  echo "เกรด A"
elif [[ $score -ge 80 ]]; then
  echo "เกรด B"
else
  echo "เกรด C"
fi

# File tests
if [[ -f "file.txt" ]]; then
  echo "file exists"
fi
# -f file  -d directory  -e exists
# -r readable  -w writable  -x executable
# -s not empty  -z string empty

# String comparison
if [[ "$name" == "admin" ]]; then
  echo "welcome admin"
fi

# Numeric: -eq -ne -gt -lt -ge -le
# Logical: && || !

🔁 Loops

# for loop
for i in 1 2 3 4 5; do
  echo "item $i"
done

# C-style for
for ((i=0; i<5; i++)); do
  echo "$i"
done

# loop ไฟล์
for file in *.txt; do
  echo "processing $file"
done

# loop arguments
for arg in "$@"; do
  echo "arg: $arg"
done
# while loop
count=0
while [[ $count -lt 5 ]]; do
  echo "count: $count"
  ((count++))
done

# อ่านทีละบรรทัดจากไฟล์
while IFS= read -r line; do
  echo "$line"
done < file.txt

# until — วนจนกว่าจะจริง
until ping -c1 google.com; do
  echo "waiting..."
  sleep 5
done
echo "network is up!"

⚙️ Functions

#!/usr/bin/env bash

# ประกาศ function
function greet() {
  local name="$1"      # local = ไม่ใช่ global
  echo "สวัสดี $name!"
  return 0            # exit code
}

# เรียกใช้
greet "สมชาย"

# Function คืนค่า (ผ่าน echo)
function get_date() {
  echo $(date +%Y-%m-%d)
}

today=$(get_date)
echo "วันที่: $today"

# ─── ตัวอย่าง Script จริง: Backup ─────────────────
function backup() {
  local src="$1"
  local dest="$2"
  local timestamp=$(date +%Y%m%d_%H%M%S)
  local archive="${dest}/backup_${timestamp}.tar.gz"

  if [[ ! -d "$src" ]]; then
    echo "Error: ไม่พบ directory $src" >&2
    return 1
  fi

  mkdir -p "$dest"
  tar -czf "$archive" "$src"
  echo "✅ Backup เสร็จ: $archive"
}

backup "/var/www" "/backup"

Cron Jobs & SSH

ตั้งเวลาทำงานอัตโนมัติ และเชื่อมต่อเซิร์ฟเวอร์ระยะไกล

⏰ Cron — Scheduled Tasks

$ crontab -e   # แก้ไข cron ของ user นี้
$ crontab -l   # ดู cron ที่ตั้งไว้
$ crontab -r   # ลบ cron ทั้งหมด

📅 Cron Syntax

* * * * *  command
│ │ │ │ └─ วันในสัปดาห์ (0=อาทิตย์, 1=จันทร์ ... 6=เสาร์)
│ │ │ └─── เดือน (1-12)
│ │ └───── วันในเดือน (1-31)
│ └─────── ชั่วโมง (0-23)
└───────── นาที (0-59)

# ตัวอย่าง crontab
0 2 * * *    /home/user/backup.sh          # ทุกวัน 02:00
*/15 * * * * /opt/check.sh                 # ทุก 15 นาที
0 9 * * 1    /scripts/report.sh            # ทุกวันจันทร์ 09:00
0 0 1 * *    /scripts/monthly.sh           # วันที่ 1 ของทุกเดือน
30 6 * * 1-5 /scripts/workday.sh           # วันทำงาน 06:30
@reboot      /scripts/startup.sh           # ตอน boot
@daily       /scripts/daily.sh             # ทุกวัน

# redirect output
0 2 * * * /backup.sh >> /var/log/backup.log 2>&1

🔐 SSH — Secure Shell

# ─── เชื่อมต่อ ─────────────────────────────────────
$ ssh user@server.com             # เชื่อมต่อ
$ ssh -p 2222 user@server.com     # port ต่างจาก 22
$ ssh -i ~/.ssh/mykey user@host   # ใช้ key

# ─── สร้าง SSH Key ─────────────────────────────────
$ ssh-keygen -t ed25519 -C "email@example.com"
# สร้าง: ~/.ssh/id_ed25519 (private) + id_ed25519.pub (public)

# ─── Copy key ไปยัง server ────────────────────────
$ ssh-copy-id user@server.com    # copy pub key
# หลังจากนี้ SSH ได้โดยไม่ต้องใส่ password

# ─── SCP — คัดลอกไฟล์ผ่าน SSH ─────────────────────
$ scp file.txt user@server:/home/user/  # ส่งไป
$ scp user@server:/var/log/app.log .    # ดึงมา
$ scp -r mydir user@server:/opt/        # recursive

# ─── rsync — sync ไฟล์ ────────────────────────────
$ rsync -avz /local/dir user@server:/remote/
$ rsync -avz --delete src/ dest/   # sync + ลบส่วนเกิน
$ rsync -avz -e ssh user@server:/remote/ /local/

# ─── SSH Config (~/.ssh/config) ────────────────────
Host myserver
    HostName 192.168.1.100
    User ubuntu
    Port 22
    IdentityFile ~/.ssh/mykey

# ใช้งาน
$ ssh myserver   # แทน ssh ubuntu@192.168.1.100

🛎️ systemd — Service Management

# ─── จัดการ Service ─────────────────────────────
$ sudo systemctl start nginx      # เริ่ม service
$ sudo systemctl stop nginx       # หยุด service
$ sudo systemctl restart nginx    # restart
$ sudo systemctl reload nginx     # reload config
$ sudo systemctl enable nginx     # เปิดตอน boot
$ sudo systemctl disable nginx    # ไม่เปิดตอน boot
$ sudo systemctl status nginx     # ดูสถานะ
$ systemctl list-units --type=service  # ดู service ทั้งหมด

# ─── Journal / Logs ──────────────────────────────
$ journalctl -u nginx             # logs ของ nginx
$ journalctl -u nginx -f          # follow real-time
$ journalctl -u nginx --since "1h ago"  # 1 ชั่วโมงที่ผ่านมา
$ journalctl -p err               # เฉพาะ error
