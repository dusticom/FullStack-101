React คืออะไร?

ทำความรู้จัก React, Virtual DOM และโครงสร้างโปรเจค

⚛️ React คืออะไร?

React คือ JavaScript Library สำหรับสร้าง User Interface (UI) พัฒนาโดย Meta (Facebook) มีจุดเด่น:

  • Component-Based — แบ่ง UI เป็นชิ้นส่วนเล็กๆ นำกลับมาใช้ซ้ำได้
  • Declarative — บอกว่าอยากได้อะไร ไม่ต้องบอกวิธีทำ
  • Virtual DOM — อัปเดต DOM อย่างชาญฉลาด เร็วกว่า DOM จริง
  • One-way Data Flow — ข้อมูลไหลจากบนลงล่าง ควบคุมง่าย

🔄 Virtual DOM ทำงานอย่างไร?

State เปลี่ยน → Re-render Virtual DOM
Diffing → เปรียบเทียบ Virtual DOM ใหม่กับเก่า
Patch DOM → อัปเดตเฉพาะส่วนที่เปลี่ยน

ผลลัพธ์: UI อัปเดตเร็วมาก ไม่ต้อง re-render ทั้งหน้า

JSX — JavaScript + XML

ไวยากรณ์พิเศษที่ทำให้เขียน HTML ใน JavaScript ได้

⚛️
JSX ไม่ใช่ HTML จริงๆ — มันคือ syntax sugar สำหรับ React.createElement() Babel จะแปลง JSX ให้เป็น JavaScript ให้อัตโนมัติ

✍️ กฎพื้นฐานของ JSX

  • ต้อง return element เดียว (ใช้ Fragment ได้)
  • แท็กต้อง ปิดเสมอ เช่น <br />
  • ใช้ className แทน class
  • ใช้ htmlFor แทน for
  • ใช้ {expression} ใส่ค่า JavaScript
  • Style ใช้ object: style={{color:'red'}}

🔁 JSX vs createElement

ก่อน JSX
React.createElement(
  'h1',
  {className:'title'},
  'Hello World'
)
ด้วย JSX
<h1 className="title">
  Hello World
</h1>

💡 ตัวอย่าง JSX ที่ใช้งานบ่อย

function Profile() {
  const name = 'สมชาย'
  const age = 25
  const isLoggedIn = true

  return (
    {/* Fragment — ไม่สร้าง element จริงใน DOM */}
    <>
      <h1>สวัสดี, {name}!</h1>
      <p>อายุ: {age} ปี</p>

      {/* Conditional Rendering */}
      {isLoggedIn && <p>ยินดีต้อนรับ!</p>}
      {isLoggedIn ? <p>เข้าสู่ระบบแล้ว</p> : <p>กรุณาเข้าสู่ระบบ</p>}

      {/* Expression ใน JSX */}
      <p>2 + 2 = {2 + 2}</p>

      {/* Inline Style */}
      <p style={{ color: '#61dafb', fontWeight: 'bold' }}>
        Styled Text
      </p>

      {/* Self-closing tag */}
      <img src="photo.jpg" alt="รูปภาพ" />
    </>
  )
}

State — ข้อมูลที่เปลี่ยนแปลงได้

จัดการข้อมูลภายใน Component ด้วย useState Hook

⚛️
เมื่อ State เปลี่ยน React จะ re-render Component นั้น และ Children ของมันอัตโนมัติ — นี่คือหัวใจของ React

useState — Hook พื้นฐานที่สุด

import { useState } from 'react'

function Counter() {
  // [ค่าปัจจุบัน, ฟังก์ชันอัปเดต] = useState(ค่าเริ่มต้น)
  const [count, setCount] = useState(0)
  const [name, setName] = useState('สวัสดี')
  const [isOn, setIsOn] = useState(false)

  return (
    <div>
      <p>นับ: {count}</p>
      <button onClick={() => setCount(count + 1)}>เพิ่ม</button>
      <button onClick={() => setCount(count - 1)}>ลด</button>
      <button onClick={() => setCount(0)}>รีเซต</button>
    </div>
  )
}

📦 State แบบ Object

const [user, setUser] = useState({
  name: 'สมชาย',
  age: 25,
  city: 'กรุงเทพ'
})

// ❌ ผิด — แก้ไขตรงๆ ไม่ trigger re-render
user.name = 'สมหญิง'

// ✅ ถูก — Spread แล้วแก้เฉพาะส่วนที่ต้องการ
setUser({ ...user, name: 'สมหญิง' })

// ✅ Functional Update (แนะนำ)
setUser(prev => ({ ...prev, age: prev.age + 1 }))

📋 State แบบ Array

const [items, setItems] = useState(['A', 'B'])

// เพิ่มรายการ
setItems([...items, 'C'])

// ลบรายการ (index 1)
setItems(items.filter((_, i) => i !== 1))

// แก้ไขรายการ
setItems(items.map((item, i) =>
  i === 0 ? 'Z' : item
))

// ❌ ห้ามใช้ push/splice โดยตรง!
items.push('C')  // ผิด! ไม่ trigger re-render

⚡ Functional Update — สำคัญมาก

// ปัญหา: Stale State ใน async operations
const [count, setCount] = useState(0)

// ❌ อาจเกิด Bug ถ้าเรียกหลายครั้งรวดเร็ว
setCount(count + 1)  // ใช้ค่า count ณ ตอน render

// ✅ ถูกต้อง — ใช้ค่าล่าสุดเสมอ
setCount(prev => prev + 1)

// ตัวอย่างกรณีที่สำคัญ
function tripleIncrement() {
  setCount(prev => prev + 1)  // 0→1
  setCount(prev => prev + 1)  // 1→2
  setCount(prev => prev + 1)  // 2→3 ✅
}

useEffect — Side Effects

จัดการ Side Effects เช่น fetch API, subscriptions, timers

ℹ️
Side Effect คือสิ่งที่ "ออกนอก" Component เช่น fetch ข้อมูล, เปลี่ยน title, subscribe event, set timer — useEffect เป็นที่สำหรับสิ่งเหล่านี้

useEffect — รูปแบบต่างๆ

import { useState, useEffect } from 'react'

function MyComponent() {
  const [data, setData] = useState(null)
  const [count, setCount] = useState(0)

  // 1. ทำงานทุกครั้งที่ Component render (ไม่แนะนำ)
  useEffect(() => {
    console.log('render ทุกครั้ง')
  })

  // 2. ทำงานครั้งเดียวตอน mount (Dependency Array ว่าง)
  useEffect(() => {
    console.log('mount ครั้งเดียว')
    fetchData()    // ✅ เหมาะสำหรับ API call ครั้งแรก
  }, [])

  // 3. ทำงานเมื่อ count เปลี่ยน
  useEffect(() => {
    console.log(`count เปลี่ยนเป็น: ${count}`)
    document.title = `Count: ${count}`
  }, [count])

  // 4. Cleanup Function — สำคัญมาก!
  useEffect(() => {
    const timer = setInterval(() => {
      console.log('tick')
    }, 1000)

    // Cleanup: ทำงานก่อน effect ครั้งต่อไป หรือตอน unmount
    return () => clearInterval(timer)  // ✅ ป้องกัน memory leak
  }, [])
}

🌐 Fetch API ใน useEffect

function UserList() {
  const [users, setUsers] = useState([])
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)

  useEffect(() => {
    let cancelled = false  // ป้องกัน race condition

    const fetchUsers = async () => {
      try {
        setLoading(true)
        const res = await fetch('https://api.example.com/users')
        const data = await res.json()
        if (!cancelled) setUsers(data)
      } catch (err) {
        if (!cancelled) setError(err.message)
      } finally {
        if (!cancelled) setLoading(false)
      }
    }

    fetchUsers()
    return () => { cancelled = true }  // Cleanup
  }, [])

  if (loading) return <p>กำลังโหลด...</p>
  if (error) return <p>Error: {error}</p>

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  )
}

Event Handling — จัดการ Event

การจัดการ Event ใน React ต่างจาก HTML ธรรมดาอย่างไร

🖱️ Event พื้นฐาน

function EventDemo() {
  const handleClick = (e) => {
    e.preventDefault()   // หยุด default action
    console.log('คลิกแล้ว!', e.target)
  }

  const handleChange = (e) => {
    console.log(e.target.value)  // ค่าใน input
  }

  const handleSubmit = (e) => {
    e.preventDefault()  // ป้องกันหน้า reload
  }

  return (
    <form onSubmit={handleSubmit}>
      <button onClick={handleClick}>คลิก</button>
      <input onChange={handleChange} />
      <input type="submit" />
    </form>
  )
}

⬆️ Lifting State Up — ส่ง Event ขึ้น Parent

// Child ส่ง event ขึ้นไปยัง Parent ผ่าน callback prop
function ChildButton({ onAdd }) {
  return (
    <button onClick={() => onAdd('รายการใหม่')}>
      เพิ่มรายการ
    </button>
  )
}

function Parent() {
  const [items, setItems] = useState([])

  const handleAdd = (newItem) => {
    setItems(prev => [...prev, newItem])
  }

  return (
    <div>
      <ChildButton onAdd={handleAdd} />
      <ul>
        {items.map((item, i) => <li key={i}>{item}</li>)}
      </ul>
    </div>
  )
}

Lists & Keys — แสดงรายการข้อมูล

การ render รายการข้อมูลและความสำคัญของ key

⚠️
key ต้องไม่ซ้ำกันใน list เดียวกัน และไม่ควรใช้ index เป็น key ถ้า list มีการเพิ่ม/ลบ/เรียงลำดับ เพราะจะทำให้ React เกิด bug ได้

📝 render รายการด้วย .map()

function ProductList() {
  const products = [
    { id: 1, name: 'แอปเปิ้ล', price: 20 },
    { id: 2, name: 'กล้วย', price: 10 },
    { id: 3, name: 'ส้ม', price: 15 },
  ]

  return (
    <ul>
      {products.map(product => (
        {/* ✅ ใช้ id ที่ unique เป็น key */}
        <li key={product.id}>
          {product.name} — {product.price} บาท
        </li>
      ))}
    </ul>
  )
}

{products.map(p => <ProductItem key={p.id} product={p} />)}

Controlled vs Uncontrolled Components และการจัดการ Form

❌ Uncontrolled (DOM จัดการ)
function BadForm() {
  const handleSubmit = (e) => {
    e.preventDefault()
    // ต้องไปดึงค่าจาก DOM
    const name = e.target.name.value
  }

    <form onSubmit={handleSubmit}>
      <input name="name" />
    </form>
  )
}
✅ Controlled (React จัดการ)
function GoodForm() {
  const [name, setName] = useState('')

  const handleSubmit = (e) => {
    e.preventDefault()
    console.log(name)  // ✅ ง่ายมาก
  }
  return (
    <form onSubmit={handleSubmit}>
      <input
        value={name}
        onChange={e => setName(e.target.value)}
      />

📋 Form ครบถ้วน — หลายประเภท Input

function FullForm() {
  const [form, setForm] = useState({
    name: '', email: '', gender: 'male', agree: false, role: 'user'
  })

  // Handler เดียวสำหรับทุก input
  const handleChange = (e) => {
    const { name, value, type, checked } = e.target
    setForm(prev => ({
      ...prev,
      [name]: type === 'checkbox' ? checked : value
    }))
  }

  const handleSubmit = (e) => {
    e.preventDefault()
    console.log(form)
  }

  return (
    <form onSubmit={handleSubmit}>
      {/* Text Input */}
      <input name="name" value={form.name} onChange={handleChange} />

      {/* Email Input */}
      <input type="email" name="email" value={form.email} onChange={handleChange} />

      {/* Radio */}
      <input type="radio" name="gender" value="male"
        checked={form.gender === 'male'} onChange={handleChange} /> ชาย

      {/* Select */}
      <select name="role" value={form.role} onChange={handleChange}>
        <option value="user">User</option>
        <option value="admin">Admin</option>
      </select>

      {/* Checkbox */}
      <input type="checkbox" name="agree"
        checked={form.agree} onChange={handleChange} /> ยอมรับเงื่อนไข

      <button type="submit" disabled={!form.agree}>ส่ง</button>
    </form>
  )
}

useContext — แชร์ข้อมูลทั่วแอป

หลีกเลี่ยง Props Drilling ด้วย Context API

⚠️
Props Drilling คือการส่ง props ผ่านหลายชั้น Component ที่ไม่ได้ใช้ข้อมูลนั้น — Context แก้ปัญหานี้ได้ แต่ควรใช้เฉพาะข้อมูลที่ "global" จริงๆ เช่น user, theme, language

useContext — 3 ขั้นตอน

import { createContext, useContext, useState } from 'react'

// ขั้นที่ 1: สร้าง Context
const ThemeContext = createContext(null)

// ขั้นที่ 2: Provider — ครอบ Component ที่ต้องการใช้ข้อมูล
function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('dark')

  const toggleTheme = () => {
    setTheme(prev => prev === 'dark' ? 'light' : 'dark')
  }

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  )
}

// ขั้นที่ 3: ใช้งานใน Component ใดก็ได้ที่อยู่ใน Provider
function ThemeButton() {
  const { theme, toggleTheme } = useContext(ThemeContext)

  return (
    <button onClick={toggleTheme}>
      โหมดปัจจุบัน: {theme}
    </button>
  )
}

// App.jsx — ครอบ Provider ที่ root
function App() {
  return (
    <ThemeProvider>
      <Header />    {/* ทุก Component ใน Provider เข้าถึง context ได้ */}
      <Main />
      <Footer />
    </ThemeProvider>
  )
}
💡
สร้าง Custom Hook สำหรับ Context เพื่อป้องกันการใช้งานนอก Provider:
const useTheme = () => useContext(ThemeContext)

useRef — อ้างอิง DOM และค่าถาวร

เข้าถึง DOM โดยตรงและเก็บค่าที่ไม่ trigger re-render

useRef — เข้าถึง DOM

import { useRef } from 'react'

function FocusInput() {
  const inputRef = useRef(null)

  const handleFocus = () => {
    // เข้าถึง DOM element โดยตรง
    inputRef.current.focus()
    inputRef.current.value = ''
  }

  return (
    <div>
      <input ref={inputRef} type="text" />
      <button onClick={handleFocus}>
        Focus Input
      </button>
    </div>
  )
}

useRef — เก็บค่าไม่ trigger re-render

function Timer() {
  const [seconds, setSeconds] = useState(0)
  const timerRef = useRef(null)

  const start = () => {
    // เก็บ timer ID ใน ref (ไม่ต้องการ re-render)
    timerRef.current = setInterval(() => {
      setSeconds(s => s + 1)
    }, 1000)
  }

  const stop = () => {
    clearInterval(timerRef.current)
  }

  return (
    <div>
      <p>{seconds} วินาที</p>
      <button onClick={start}>เริ่ม</button>
      <button onClick={stop}>หยุด</button>
    </div>
  )
}

useMemo & useCallback — Performance

Memoization เพื่อป้องกันการคำนวณซ้ำโดยไม่จำเป็น

⚠️
อย่าใช้ useMemo/useCallback ทุกที่! ใช้เฉพาะเมื่อมีปัญหา performance จริงๆ เพราะ Memoization เองก็มีค่าใช้จ่าย

useMemo — Cache ผลลัพธ์การคำนวณ

import { useMemo, useState } from 'react'

function ExpensiveCalc({ numbers }) {
  const [filter, setFilter] = useState('')

  // คำนวณใหม่เฉพาะเมื่อ numbers เปลี่ยน
  const total = useMemo(() => {
    console.log('คำนวณ...')
    return numbers.reduce((sum, n) => sum + n, 0)
  }, [numbers])   // Dependency

  // กรอง list — คำนวณใหม่เมื่อ numbers หรือ filter เปลี่ยน
  const filtered = useMemo(() =>
    numbers.filter(n => n.toString().includes(filter))
  , [numbers, filter])

  return <div>ผลรวม: {total}</div>
}

useCallback — Cache ฟังก์ชัน

import { useCallback, useState } from 'react'

function Parent() {
  const [count, setCount] = useState(0)
  const [text, setText] = useState('')

  // ไม่สร้างฟังก์ชันใหม่ทุกครั้ง Parent re-render
  // สร้างใหม่เฉพาะเมื่อ count เปลี่ยน
  const handleAdd = useCallback(() => {
    setCount(c => c + 1)
  }, [])  // ไม่มี dependency → สร้างครั้งเดียว

  return (
    <div>
      <input onChange={e => setText(e.target.value)} />
      {/* Child ไม่ re-render เมื่อ text เปลี่ยน */}
      <ExpensiveChild onAdd={handleAdd} />
    </div>
  )
}

Custom Hooks — นำ Logic กลับมาใช้ซ้ำ

สร้าง Hook ของตัวเองเพื่อแยก Logic ออกจาก UI

💡
Custom Hook คือฟังก์ชัน JavaScript ปกติที่ชื่อขึ้นต้นด้วย "use" และสามารถเรียก Hook อื่นๆ ได้ — ทำให้แชร์ Logic ระหว่าง Component ได้โดยไม่ต้อง copy โค้ด

🔧 useFetch — Fetch ข้อมูลจาก API

// hooks/useFetch.js
import { useState, useEffect } from 'react'

export function useFetch(url) {
  const [data, setData] = useState(null)
  const [loading, setLoading] = useState(true)
  const [error, setError] = useState(null)

  useEffect(() => {
    let cancelled = false
    setLoading(true)

    fetch(url)
      .then(res => res.json())
      .then(data => { if (!cancelled) { setData(data); setLoading(false) } })
      .catch(err => { if (!cancelled) { setError(err.message); setLoading(false) } })

    return () => { cancelled = true }
  }, [url])

  return { data, loading, error }
}

// ใช้งานใน Component — สะอาดมาก!
function UserProfile({ userId }) {
  const { data: user, loading, error } = useFetch(`/api/users/${userId}`)

  if (loading) return <p>กำลังโหลด...</p>
  if (error) return <p>Error: {error}</p>
  return <h1>{user.name}</h1>
}

🔧 useLocalStorage

function useLocalStorage(key, initial) {
  const [value, setValue] = useState(() => {
    try {
      const saved = localStorage.getItem(key)
      return saved ? JSON.parse(saved) : initial
    } catch {
      return initial
    }
  })

  const setStored = (newVal) => {
    localStorage.setItem(key, JSON.stringify(newVal))
    setValue(newVal)
  }

  return [value, setStored]
}

// ใช้งาน
const [theme, setTheme] = useLocalStorage('theme', 'dark')

🔧 useDebounce

function useDebounce(value, delay = 500) {
  const [debounced, setDebounced] = useState(value)

  useEffect(() => {
    const timer = setTimeout(() => {
      setDebounced(value)
    }, delay)

    return () => clearTimeout(timer)
  }, [value, delay])

  return debounced
}

// ใช้งาน — ค้นหาหลังหยุดพิมพ์ 500ms
function Search() {
  const [query, setQuery] = useState('')
  const debounced = useDebounce(query, 500)

  useEffect(() => {
    if (debounced) fetchSearch(debounced)
  }, [debounced])  // เรียก API เฉพาะค่า debounced
}

Patterns & Best Practices

รูปแบบที่ดีในการเขียน React และ Conditional Rendering

🎨 Conditional Rendering รูปแบบต่างๆ

function Dashboard({ isLoggedIn, role, count, error }) {
  // 1. Early Return
  if (!isLoggedIn) return <Login />

  return (
    <div>
      {/* 2. && operator */}
      {error && <ErrorBanner message={error} />}

      {/* 3. Ternary */}
      {role === 'admin' ? <AdminPanel /> : <UserPanel />}

      {/* 4. Nullish Coalescing */}
      {count ?? <p>ไม่มีข้อมูล</p>}

      {/* 5. Switch ใน function */}
      {renderStatus(role)}
    </div>
  )
}

// Switch แบบแยกฟังก์ชัน — ชัดเจนกว่า nested ternary
function renderStatus(role) {
  switch (role) {
    case 'admin': return <span>👑 Admin</span>
    case 'mod': return <span>🛡️ Moderator</span>
    default: return <span>👤 User</span>
  }
}

✅ Best Practices สรุป

  • ใช้ Functional Components เสมอ (ไม่ใช้ Class)
  • ชื่อ Component ขึ้นต้นด้วยตัวพิมพ์ใหญ่
  • 1 Component ต่อ 1 ไฟล์ (Single Responsibility)
  • State ไว้ต่ำที่สุดเท่าที่จำเป็น (Lift only when needed)
  • ใช้ Functional Update เมื่อ state ขึ้นกับค่าเก่า
  • เสมอ Cleanup ใน useEffect
  • ใช้ key ที่ unique ใน list
  • ไม่แก้ไข state โดยตรง (Immutability)
  • แยก Logic ออกเป็น Custom Hooks
  • ใช้ useMemo/useCallback เฉพาะเมื่อจำเป็น
